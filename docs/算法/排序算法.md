# 快速排序
思路：选择一个基准pivot，把小于pivot的放左边，大于pivot的放右边。
<div></div>
每次可以粗略的把数组分为小于基准leftArr、大于基准rightArr两块，然后递归leftArr、rightArr，直到数组数量小于3，进行最后一次精准的排序，就实现了整个数组的排序。
<div></div>

* **时间复杂度**，
    * 理想情况下每次都均分数组，要logn次partition，每次partition扫描一遍耗费n，时间复杂度是 O(nlogn)；
    * 在最坏的情况下，pivot每次选择了最大/最小值，导致数组都被分配到了一边，就需要n次partion，因此时间复杂度 O(n ^ 2)。尽管可以通过随机数选择pivot降低每次都选到极值的概率，但理论上，无法避免这种情况
* **空间复杂度**，原地排序，没有创建额外的新数组，因此空间复杂度 O(1)
* **稳定性**，在partion过程中，会直接跨距离交换位置，而不是按顺序搬运，因此相等的元素相对顺序会被破坏，因此不稳定。
    * 以[2A, 1, 2B]为例，2A、2B是指位置不同的2，以2B为基准，排序后就是[1, 2B, 2A]，顺序被破坏

```js
var sortArray = function (nums) {
    /**
     * @description：把数组分为小于基准、基准、大于基准
     * @return：返回基准索引，供后续递归使用
     */
    const partition = (left, right) => {
        // 用以比较的基准，如果是极值，会有把数组都分到一边的风险
        const pivot = nums[right];
        // i左边用来存小于pivot的值，移动的做法和Leetcode 283.移动零相似
        let i = left;
        for (let j = left; j < right; j++) {
            if (nums[j] < pivot) {
                [nums[j], nums[i]] = [nums[i], nums[j]];
                i++;
            }
        }

        // 最后，i是第一个大于pivot的值，因此把pivot放在i这个位置，nums[i]>pivot，放在right的位置
        [nums[i], nums[right]] = [nums[right], nums[i]];
        return i;
    }

    const quickSort = (left, right) => {
        if (left >= right) {
            return;
        }
        const pivotIndex = partition(left, right);
        quickSort(0, pivotIndex - 1);
        quickSort(pivotIndex + 1, right);
    }

    quickSort(0, nums.length - 1);
    return nums;
};
```

# 归并排序
思路：合并两个有序数组，把数组不断二分，直到只有一个元素，然后再两两合并成一个新的有序数组，最后完成整个数组的排序
* **时间复杂度**：O(nlogn)。每次均匀的二分数组，因此稳定的经历logn次merge，每次merge，需要遍历当前的数组，因此，时间复杂度是固定的O(nlogn)
* **空间复杂度**：O(n)。每次merge都创建一个数组，但是merge完空间就释放了，因此同一时间只额外创建了一个数组，空间复杂度为O(n)
* **稳定性**：稳定排序。合并的过程，如果相等，优先取左边，可以保证相等元素的相对顺序不变，因此，是稳定排序

```js
var sortArray = function (nums) {
    // 合并两个有序数组
    const merge = (leftArr, rightArr) => {
        const res = [];
        let left = 0;
        let right = 0;

        while (left < leftArr.length && right < rightArr.length) {
            // 相等时，优先取左边，保证稳定性
            if (leftArr[left] <= rightArr[right]) {
                res.push(leftArr[left]);
                left++;
            } else {
                res.push(rightArr[right]);
                right++;
            }
        }

        if (left < leftArr.length) {
            res.push(...leftArr.slice(left));
        } else {
            res.push(...rightArr.slice(right));
        }

        return res;
    }

    const mergeSort = (arr) => {
        if (arr.length <= 1) {
            return arr;
        }
        const mid = Math.floor(arr.length / 2);
        const leftArr = mergeSort(arr.slice(0, mid));
        const rightArr = mergeSort(arr.slice(mid));
        return merge(leftArr, rightArr);
    }

    return mergeSort(nums);
};
```

由于链表无法快速访问索引，快速排序中的分割，交换都难以实现，而归并排序的合并不受影响，因此归并排序非常适合用在排序链表的场景中。
链表排序代码如下：
```js
/**
 * Definition for singly-linked list.
 * function ListNode(val, next) {
 *     this.val = (val===undefined ? 0 : val)
 *     this.next = (next===undefined ? null : next)
 * }
 */

/**
 * @param {ListNode} head
 * @return {ListNode}
 */
var sortList = function (head) {
    if (!head || !head.next) {
        return head;
    }
    // 从中间断开，返回右边的第一个节点
    const splitFromMiddle = (node) => {
        if (!node || !node.next) {
            return node;
        }
        let slow = node;
        let fast = node.next;
        while (fast && fast.next) {
            slow = slow.next;
            fast = fast.next.next;
        }
        const next = slow.next;
        slow.next = null;
        return next;
    }

    const merge = (l1, l2) => {
        const preHead = new ListNode(0);
        let cur = preHead;
        while (l1 && l2) {
            if (l1.val < l2.val) {
                cur.next = l1;
                l1 = l1.next;
            } else {
                cur.next = l2;
                l2 = l2.next;
            }
            cur = cur.next;
        }
        cur.next = l1 || l2;
        return preHead.next;
    }

    let head2 = splitFromMiddle(head);
    head = sortList(head);
    head2 = sortList(head2);
    return merge(head, head2);
};
```

# 插入排序
思路：i是正要排序的元素，i左边是已经有序的数组，每次遍历试图把nums[i]放在恰当的顺序中
* **时间复杂度**：O(n ^ 2)，双重遍历
    * 在“几乎有序”的数组中极为高效，例如[1, 2, 3, 4, 5, 6, 7, 8, 0]，只需要把0移动到首位，接近O(n)的复杂度。因此，在js中内置的sort，在数组容量较小时，使用插入排序
* **空间复杂度**：O(1)，无额外空间消耗
* **稳定性**：稳定，每次排序，只把元素向右移动，不会破坏相等元素的相对顺序
```js
var sortArray = function (nums) {
    // i左边是已经排序好的数组，每次排序把nums[i]插入到左边
    for (let i = 1; i < nums.length; i++) {
        let key = nums[i];
        let j = i - 1;

        // 更大的数字往右移动，给nums[i]腾出位置
        while (j >= 0 && nums[j] > key) {
            nums[j + 1] = nums[j];
            j--;
        }

        // j + 1，就是nums[i]该在的位置
        nums[j + 1] = key;
    }
    return nums;
};
```
