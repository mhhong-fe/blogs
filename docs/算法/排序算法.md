# 排序算法

这篇笔记主要整理三种高频排序：**快速排序**、**归并排序**、**插入排序**。如果只是想先抓住重点，可以先看下面这张表：

| 算法 | 平均时间复杂度 | 空间复杂度 | 稳定性 | 适合场景 |
| --- | --- | --- | --- | --- |
| 快速排序 | `O(nlogn)` | `O(1)` | 不稳定 | 原地排序、追求平均性能 |
| 归并排序 | `O(nlogn)` | `O(n)` | 稳定 | 需要稳定排序，或处理链表 |
| 插入排序 | `O(n^2)` | `O(1)` | 稳定 | 数据量小，或数组基本有序 |

如果你只想记一个结论：

- 想要**平均性能好**，优先看快速排序。
- 想要**稳定排序**，优先看归并排序。
- 数据量小，或者数组已经**接近有序**，插入排序往往很顺手。

# 快速排序

> 核心思路：选一个基准值 `pivot`，把小于它的放到左边，大于等于它的放到右边，再分别递归处理左右两部分。

快速排序的关键在于 `partition`。每轮 `partition` 结束后，`pivot` 都会落到最终位置上，接着继续处理它左右两侧的区间，直到区间长度为 `0` 或 `1`。

- **时间复杂度**
  - 平均情况：`O(nlogn)`。每轮大致把数组分成两半，要递归 `logn` 层，每层扫描一次区间。
  - 最坏情况：`O(n^2)`。如果每次都选到最大值或最小值，数组会严重倾斜，只能一边一边地拆。
- **空间复杂度**：`O(1)`。下面这版是原地排序，不额外创建新数组。
- **稳定性**：**不稳定**。`partition` 过程中会发生跨位置交换，相等元素的相对顺序可能被打乱。
  - 例如 `[2A, 1, 2B]`，如果以 `2B` 为基准，排序后可能变成 `[1, 2B, 2A]`。

```js
var sortArray = function (nums) {
    /**
     * 把当前区间分成：小于 pivot | pivot | 大于等于 pivot
     * 返回 pivot 的最终索引，供后续递归使用
     */
    const partition = (left, right) => {
        // 这里直接取区间右端点作为基准值
        const pivot = nums[right];
        // [left, i) 始终维护为小于 pivot 的区间
        let i = left;

        for (let j = left; j < right; j++) {
            if (nums[j] < pivot) {
                [nums[i], nums[j]] = [nums[j], nums[i]];
                i++;
            }
        }

        // 把 pivot 放到中间位置
        [nums[i], nums[right]] = [nums[right], nums[i]];
        return i;
    };

    const quickSort = (left, right) => {
        if (left >= right) {
            return;
        }

        const pivotIndex = partition(left, right);
        quickSort(left, pivotIndex - 1);
        quickSort(pivotIndex + 1, right);
    };

    quickSort(0, nums.length - 1);
    return nums;
};
```

快速排序的理解重点可以浓缩成一句话：**先把一个元素放到最终位置，再递归解决剩下的问题。**

# 归并排序

> 核心思路：先拆分，再合并。把数组不断二分，直到每段只剩一个元素，再把两个有序数组合并成一个更大的有序数组。

归并排序的重点不是“分”，而是“合”。只要合并两个有序数组这一步写对了，整个排序过程就比较稳定。

- **时间复杂度**：`O(nlogn)`。无论数据分布如何，都会经历 `logn` 层拆分，每层合并总共处理 `n` 个元素。
- **空间复杂度**：`O(n)`。合并时需要额外数组保存结果。
- **稳定性**：**稳定**。如果两个元素相等，优先取左边元素，就能保留原来的相对顺序。

```js
var sortArray = function (nums) {
    // 合并两个有序数组
    const merge = (leftArr, rightArr) => {
        const res = [];
        let left = 0;
        let right = 0;

        while (left < leftArr.length && right < rightArr.length) {
            // 相等时优先取左边，保证稳定性
            if (leftArr[left] <= rightArr[right]) {
                res.push(leftArr[left]);
                left++;
            } else {
                res.push(rightArr[right]);
                right++;
            }
        }

        if (left < leftArr.length) {
            res.push(...leftArr.slice(left));
        } else {
            res.push(...rightArr.slice(right));
        }

        return res;
    };

    const mergeSort = (arr) => {
        if (arr.length <= 1) {
            return arr;
        }

        const mid = Math.floor(arr.length / 2);
        const leftArr = mergeSort(arr.slice(0, mid));
        const rightArr = mergeSort(arr.slice(mid));
        return merge(leftArr, rightArr);
    };

    return mergeSort(nums);
};
```

链表不支持随机访问，下标操作和交换节点都不方便，所以快速排序在链表场景里不太好写。归并排序只需要“找中点 + 合并两个有序链表”，因此更适合链表排序。

```js
/**
 * Definition for singly-linked list.
 * function ListNode(val, next) {
 *     this.val = (val === undefined ? 0 : val);
 *     this.next = (next === undefined ? null : next);
 * }
 */

/**
 * @param {ListNode} head
 * @return {ListNode}
 */
var sortList = function (head) {
    if (!head || !head.next) {
        return head;
    }

    // 从中间断开链表，返回右半部分头结点
    const splitFromMiddle = (node) => {
        if (!node || !node.next) {
            return node;
        }

        let slow = node;
        let fast = node.next;

        while (fast && fast.next) {
            slow = slow.next;
            fast = fast.next.next;
        }

        const next = slow.next;
        slow.next = null;
        return next;
    };

    const merge = (l1, l2) => {
        const preHead = new ListNode(0);
        let cur = preHead;

        while (l1 && l2) {
            if (l1.val <= l2.val) {
                cur.next = l1;
                l1 = l1.next;
            } else {
                cur.next = l2;
                l2 = l2.next;
            }
            cur = cur.next;
        }

        cur.next = l1 || l2;
        return preHead.next;
    };

    let head2 = splitFromMiddle(head);
    head = sortList(head);
    head2 = sortList(head2);
    return merge(head, head2);
};
```

归并排序最值得记住的是它的“稳定性来源”：**合并时，相等元素优先取左边。**

# 插入排序

> 核心思路：把数组左侧看成“已经排好序”的部分，每次取一个新元素，插入到它该在的位置。

插入排序写法简单，但性能比较依赖数据分布。数据量不大、或者数组本来就“差不多有序”时，它反而很实用。

- **时间复杂度**：`O(n^2)`。最常见的写法需要双重遍历。
  - 如果数组本身接近有序，例如 `[1, 2, 3, 4, 5, 6, 7, 8, 0]`，移动次数会明显变少，实际表现会更接近 `O(n)`。
- **空间复杂度**：`O(1)`。
- **稳定性**：**稳定**。它只是把更大的元素依次右移，不会打乱相等元素的相对顺序。

```js
var sortArray = function (nums) {
    // nums[0..i-1] 始终是有序区间
    for (let i = 1; i < nums.length; i++) {
        const key = nums[i];
        let j = i - 1;

        // 把比 key 大的元素整体右移
        while (j >= 0 && nums[j] > key) {
            nums[j + 1] = nums[j];
            j--;
        }

        // j + 1 就是 key 应该插入的位置
        nums[j + 1] = key;
    }

    return nums;
};
```

插入排序可以把思路想得更形象一点：**像打扑克牌理牌一样，每拿到一张新牌，就插到左边合适的位置。**

# 怎么选

- 如果题目强调 **原地排序**，通常先考虑快速排序。
- 如果题目强调 **稳定性**，或者对象里有“主键相同但还要保留原顺序”的要求，优先考虑归并排序。
- 如果是 **链表排序**，归并排序基本是更自然的选择。
- 如果数据规模不大，或者输入本来就接近有序，插入排序实现简单，实际表现也不差。
