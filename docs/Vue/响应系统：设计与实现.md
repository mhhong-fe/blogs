# 响应式系统设计与实现

## 一、什么是响应式数据

当某个数据发生变化时，所有依赖该数据的副作用函数（如 DOM 更新、模板渲染等）能够自动重新执行，这种数据称为**响应式数据**。在 Vue 3 中，通常通过 `reactive`、`ref`、`computed`、`watch` 等 API 来创建和管理响应式数据。

## 二、渐进式实现

### 1. 基础实现

核心思路如下：

1. 读取操作发生时，将当前副作用函数收集到依赖集合（"桶"）中；
2. 设置操作发生时，从依赖集合中取出所有副作用函数并依次执行。

Vue 3 使用 `Proxy` 拦截对象的读写操作，最基础的响应式实现如下：

```js
// 存储副作用函数的桶
const bucket = new Set();

// 原始数据
const data = { text: "hello world" };
// 对原始数据的代理
const obj = new Proxy(data, {
    // 拦截读取操作
    get(target, key) {
        // 将副作用函数 effect 添加到存储副作用函数的桶中
        bucket.add(effect);
        // 返回属性值
        return target[key];
    },
    // 拦截设置操作
    set(target, key, newVal) {
        // 设置属性值
        target[key] = newVal;
        // 把副作用函数从桶里取出并执行
        bucket.forEach((fn) => fn());
        // 返回 true 代表设置操作成功
        return true;
    },
});

// 副作用函数
function effect() {
  document.body.innerText = obj.text
}

setTimeout(() => {
    obj.text = "hello vue3";
}, 1000);
```

### 2. 统一注册副作用函数

> 在 Vue 中，模板编译后的 render 函数、computed、watch 等 API，均基于 effect 函数实现。

上述实现中，副作用函数硬编码为 `effect`，实际场景中副作用函数可以是任意名称，也可以是匿名函数。因此，需要一套统一的副作用函数注册机制，使任意函数（包括匿名函数）都能被正确收集。

方案是引入 `effect` 注册函数，将所有副作用函数统一赋值给全局变量 `activeEffect`，依赖收集时存储 `activeEffect` 即可。

```js
// 用一个全局变量存储当前激活的副作用函数
let activeEffect;
function effect(fn) {
    activeEffect = fn;
    // 执行副作用函数以触发依赖收集（render 函数注册时立即执行）
    fn();
}

// 注册一个匿名副作用函数
effect(() => {
    console.log("effect run");
    document.body.innerText = obj.text;
});
```

对应的依赖收集逻辑更新为：

```js
// 存储副作用函数的桶
const bucket = new Set();

const data = { text: "hello world" };
const obj = new Proxy(data, {
    get(target, key) {
        // 收集当前激活的副作用函数
        bucket.add(activeEffect);
        return target[key];
    },
    set(target, key, newVal) {
        target[key] = newVal;
        bucket.forEach((fn) => fn());
    },
});
```

### 3. 细化依赖粒度

当前依赖的粒度停留在对象层面：副作用函数 `effectFn` 依赖 `obj.text`，但 `obj.name` 改变同样会触发 `effectFn` 执行，这是不必要的开销。因此需要将依赖粒度细化到属性层面。

使用 `WeakMap` 构建 `target → (key → effectFns)` 的树状存储结构。选用 `WeakMap` 的原因是：当 `target` 不再被引用时，对应的 key 和 effectFns 可被垃圾回收，避免内存泄漏。

![依赖存储结构](https://cdn.polyspeak.ai/speakmaster/edd5b03582f5faefdccbcf662718dd25.webp)

实现要点：

- 使用 `WeakMap` 替代 `Set` 存储副作用函数
- 封装依赖收集函数 `track`
- 封装依赖触发函数 `trigger`

```js
const bucket = new WeakMap()

const data = { text: 'hello world' }
const obj = new Proxy(data, {
  get(target, key) {
    track(target, key)
    return target[key]
  },
  set(target, key, newVal) {
    target[key] = newVal
    trigger(target, key)
  }
})

function track(target, key) {
  let depsMap = bucket.get(target)
  if (!depsMap) {
    bucket.set(target, (depsMap = new Map()))
  }
  let deps = depsMap.get(key)
  if (!deps) {
    depsMap.set(key, (deps = new Set()))
  }
  deps.add(activeEffect)
}

function trigger(target, key) {
  const depsMap = bucket.get(target)
  if (!depsMap) return
  const effects = depsMap.get(key)
  effects && effects.forEach(fn => fn())
}

let activeEffect
function effect(fn) {
  activeEffect = fn
  fn()
}

effect(() => {
  console.log('effect run')
  document.body.innerText = obj.text
})

setTimeout(() => {
  obj.text = 'hello vue3'
}, 1000)
```

> 对于深层嵌套属性，Proxy 会递归代理访问路径上的属性。例如访问 `state.info.age` 时，`state.info` 也会被代理，因此修改 `state.info.age` 不会影响依赖 `state.info.name` 的副作用函数。

### 4. 分支切换与 cleanup

考虑如下场景：

```js
const data = { ok: true, text: "hello world" };
const obj = new Proxy(data, { /* ... */ });

effect(function effectFn() {
    document.body.innerText = obj.ok ? obj.text : "not";
});
```

- 当 `obj.ok` 为 `true` 时，`effectFn` 应进入 `obj.text` 的依赖集合；
- 当 `obj.ok` 变为 `false` 后，`obj.text` 的变动不应再触发 `effectFn`，因此需要从 `obj.text` 的依赖集合中将其移除。

实现思路：每次副作用函数执行前，先将其从所有依赖集合中删除；执行期间，再根据实际读取的属性重新建立依赖关系（在 `track` 中完成）。

为此，需要维护副作用函数与依赖集合的双向映射：

![双向映射关系](https://cdn.polyspeak.ai/speakmaster/17605c56333b9d37bbdac81f5db07279.webp)

`effect` 与 `cleanup` 实现：

```js
let activeEffect
function effect(fn) {
  const effectFn = () => {
    cleanup(effectFn)
    activeEffect = effectFn
    fn()
  }
  // effectFn.deps 存储所有包含该副作用函数的依赖集合
  effectFn.deps = []
  effectFn()
}

function cleanup(effectFn) {
  for (let i = 0; i < effectFn.deps.length; i++) {
    const deps = effectFn.deps[i]
    deps.delete(effectFn)
  }
  effectFn.deps.length = 0
}
```

`track` 中同步建立反向引用：

```js
function track(target, key) {
  let depsMap = bucket.get(target)
  if (!depsMap) {
    bucket.set(target, (depsMap = new Map()))
  }
  let deps = depsMap.get(key)
  if (!deps) {
    depsMap.set(key, (deps = new Set()))
  }
  deps.add(activeEffect)
  // 将当前依赖集合加入副作用函数的 deps，建立反向引用
  activeEffect.deps.push(deps)
}
```

`trigger` 中需要避免遍历时同时增删导致的无限循环。ECMAScript 规范规定，在 `forEach` 遍历 Set 期间，若某个值被删除后又重新添加，该值将被再次访问，从而引发无限循环。解决方案是将待执行的副作用函数复制到一个新 Set 中再遍历：

```js
// 直接遍历 effects 的问题：cleanup 会删除其中的元素，
// effectFn 执行时 track 又会重新添加，导致 forEach 无限循环。
// effects && effects.forEach(effectFn => effectFn())  // ❌

// 正确做法：先将待执行集合复制到新 Set，再遍历执行
function trigger(target, key) {
  const depsMap = bucket.get(target)
  if (!depsMap) return
  const effects = depsMap.get(key)
  const effectsToRun = new Set()
  effects && effects.forEach(effectFn => {
    // 避免副作用函数自身触发自身，防止无限递归
    if (effectFn !== activeEffect) {
      effectsToRun.add(effectFn)
    }
  })
  effectsToRun.forEach(effectFn => effectFn())
}
```

无限循环问题的最小复现：

```js
// 直接在遍历 set 自身时，对其执行 delete 后再 add，
// 被删除后重新添加的元素会排到末尾并被再次访问，导致无限循环
const set = new Set([1])
set.forEach(item => {
  set.delete(1)
  set.add(1)
  console.log(999) // 无限打印
})
```

### 5. 支持嵌套 effect

组件的嵌套渲染是嵌套 effect 的典型场景：

```js
effect(() => {
    Foo.render();
    effect(() => {
        Bar.render();
    });
});
```

当前 `activeEffect` 是单一全局变量，执行内层 effect 时 `activeEffect` 被覆盖为内层函数，内层执行完毕后 `activeEffect` 无法还原，导致外层函数对应的属性错误地收集了内层副作用函数。

解决方案是引入**副作用函数栈**：副作用函数执行前压栈，执行完毕后出栈，`activeEffect` 始终指向栈顶元素，从而保证每个响应式属性只收集直接读取它的副作用函数。

![effectStack 示意](https://cdn.polyspeak.ai/speakmaster/e79452b4d7c022f44b07b258cf9cf957.webp)

```js
let activeEffect
const effectStack = []

function effect(fn) {
  const effectFn = () => {
    cleanup(effectFn)
    activeEffect = effectFn
    effectStack.push(effectFn)
    fn()
    effectStack.pop()
    activeEffect = effectStack[effectStack.length - 1]
  }
  effectFn.deps = []
  effectFn()
}
```

### 6. 执行时机调度

当前副作用函数在属性变更后立即同步执行，但实际框架中（如组件渲染）通常需要在本轮同步任务结束后批量执行副作用，因此需要支持对执行时机的调度。

为 `effect` 增加 `options.scheduler` 参数，`trigger` 触发时若存在调度器，则由调度器决定执行方式：

```js
function effect(fn, options = {}) {
  const effectFn = () => {
    cleanup(effectFn)
    activeEffect = effectFn
    effectStack.push(effectFn)
    fn()
    effectStack.pop()
    activeEffect = effectStack[effectStack.length - 1]
  }
  effectFn.options = options
  effectFn.deps = []
  effectFn()
}

function trigger(target, key) {
  const depsMap = bucket.get(target)
  if (!depsMap) return
  const effects = depsMap.get(key)

  const effectsToRun = new Set()
  effects && effects.forEach(effectFn => {
    if (effectFn !== activeEffect) {
      effectsToRun.add(effectFn)
    }
  })
  effectsToRun.forEach(effectFn => {
    if (effectFn.options.scheduler) {
      effectFn.options.scheduler(effectFn)
    } else {
      effectFn()
    }
  })
}
```

通过 `scheduler` 将副作用延迟到宏任务中执行：

```js
effect(() => console.log(obj.text), {
    scheduler(fn) {
        setTimeout(fn)
    }
})
```

### 7. 执行次数调度

连续多次修改同一响应式数据时，副作用函数只需根据最终状态执行一次：

```js
obj.foo++;
obj.foo++;
// 期望：effect 只执行一次，输出最终值
```

利用微任务队列实现去重与批量刷新：

```js
// 任务队列（Set 保证同一副作用函数不重复入队）
const jobQueue = new Set();
const p = Promise.resolve();

let isFlushing = false;
function flushJob() {
    if (isFlushing) return;
    isFlushing = true;
    p.then(() => {
        jobQueue.forEach((job) => job());
    }).finally(() => {
        isFlushing = false;
    });
}

effect(
    () => { console.log(obj.foo); },
    {
        scheduler(fn) {
            jobQueue.add(fn);
            flushJob();
        },
    }
);

obj.foo++;
obj.foo++;
```

执行流程：

- 每次写操作触发 `scheduler`，将副作用函数加入 `jobQueue` 并调用 `flushJob`；
- `jobQueue` 为 Set 结构，相同副作用函数不会重复入队；
- `Promise.then` 将队列刷新推迟到当前同步任务全部完成后执行；
- `isFlushing` 标志确保一次事件循环中队列只被刷新一次。

### 8. lazy 与返回值

`computed` 和 `watch` 要求 effect 支持两项扩展：

- **`lazy` 选项**：注册时不立即执行副作用函数，由外部在合适时机手动调用；
- **返回值透传**：`effectFn` 将 `fn()` 的执行结果返回，`computed` 需要以此拿到 getter 的计算值。

```js
function effect(fn, options = {}) {
  const effectFn = () => {
    cleanup(effectFn)
    activeEffect = effectFn
    effectStack.push(effectFn)
    const res = fn()  // 捕获返回值
    effectStack.pop()
    activeEffect = effectStack[effectStack.length - 1]
    return res        // 透传给调用方
  }
  effectFn.options = options
  effectFn.deps = []
  if (!options.lazy) {
    effectFn()
  }
  return effectFn     // 返回 effectFn，供 lazy 场景手动调用
}
```

同时，`track` 增加守卫：若当前没有激活的副作用函数，直接返回，避免无关读取写入依赖桶。

```js
function track(target, key) {
  if (!activeEffect) return
  let depsMap = bucket.get(target)
  if (!depsMap) bucket.set(target, (depsMap = new Map()))
  let deps = depsMap.get(key)
  if (!deps) depsMap.set(key, (deps = new Set()))
  deps.add(activeEffect)
  activeEffect.deps.push(deps)
}
```

### 9. 使用 Reflect 解决 this 指向问题

直接通过 `target[key]` 返回属性值存在 `this` 指向问题：

```js
const obj = {
    foo: 1,
    get bar() {
        return this.foo;
    },
};

const p = new Proxy(obj, {
    get(target, key) {
        track(target, key);
        return target[key];
    },
});

effect(() => {
    console.log(p.bar);
});

p.foo++;
```

访问 `p.bar` 时，getter 内的 `this` 指向原始对象 `target`（即 `obj`），而非代理对象 `p`，导致 `this.foo` 的读取未经过 Proxy 拦截，`track` 未被调用，`p.foo++` 因此无法触发 effect 重新执行。

使用 `Reflect.get` 并传入 `receiver` 可将 `this` 正确绑定为代理对象：

```js
const p = new Proxy(obj, {
    get(target, key, receiver) {
        track(target, key);
        // receiver 即代理对象 p，getter 内的 this 将指向 p
        return Reflect.get(target, key, receiver);
    },
});
```

## 三、总结

本文基于《Vue.js 设计与实现》中响应系统一章，从最简实现出发，逐步完善了以下机制：

| 章节 | 解决的问题 |
|------|-----------|
| 统一注册副作用函数 | 支持任意名称及匿名函数 |
| 细化依赖粒度 | 依赖精确到属性级别，避免冗余触发 |
| 分支切换与 cleanup | 动态维护依赖关系，避免无效执行 |
| 嵌套 effect 支持 | effectStack 保证多层嵌套下依赖收集的正确性 |
| 执行时机调度 | scheduler 支持异步延迟执行 |
| 执行次数调度 | 微任务队列去重，实现批量更新 |
| lazy 与返回值 | 支持懒执行与结果获取，为 computed/watch 提供基础 |
| Reflect 修正 this | 确保 getter 内属性访问经过 Proxy 拦截 |
